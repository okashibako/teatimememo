■コマンド内での作業
①今あるnode_moduleを削除。削除後にnpm iを実行しライブラリを入れる

先にcordova plugin rm cordova-plugin-exif

cordova plugin add platforms(exifがないと見なされ途中でインストールが終わってしまうのでrmした後に実行する)

cordova plugin add https://github.com/domax/cordova-exif.git --force

npm run build

imgcache.jsをimgcacheにフォルダ名変更

cordova prepare android

①build.gradleのwrapperでエラーになっているところ
wrapper {
    gradleVersion = '2.14.1'
}
のような形にする

②build.gradleの「task cdvPrintProps <<」の<<を消去し

doLast{println('cdvCompileSdkVersion=' + cdvCompileSdkVersion)...}で囲む

③build.gradleのdependencies内のclasspath 'com.android.tools.build:gradle:2.2.3'を

classpath 'com.android.tools.build:gradle:3.5.1'に変更してsync

④マニフェストに記述してあるtargetSdkVersion 28とminSdkVersion 15を

build.gradleに移す（左下にでてくるdo Refuctaringでできる）

defaultConfig {
targetSdkVersion 28
        minSdkVersion 15
}

compileSdkVersion 28
buildToolsVersion "28.0.3"


⑤dependencies内の不用な二つを削除して新しくimplementation project(":CordovaLib")を追加

⑥dependencies内のライブラリのバージョンを以下のように合わせる

implementation "com.android.support:support-v4:24.1.1+"
implementation "com.android.support:appcompat-v7:26.1.0"
implementation "com.android.support:design:26.1.0"
implementation "com.android.support:cardview-v7:26.1.0"
implementation "com.google.firebase:firebase-core:16.0.8"
implementation "com.google.firebase:firebase-messaging:17.5.0"

※firebase系は+のままだと最新のを引っ張ってきてしまうため上記のバージョンに更新する

⑦res\values内にcolor.xmlというxmlファイルを作る↓

<?xml version="1.0" encoding="utf-8"?>
<resources>
    <color name="hint_foreground_material_light">#FFFFFF</color>
    <color name="hint_foreground_material_dark">#000000</color>
</resources>
